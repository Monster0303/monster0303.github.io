一、Ansible 是什么¶

在生产环境中，会有很多集群，比如 openstack 集群、ceph 集群、k8s 集群等等，集群中的主机数量从十几台到上百、上千台不等，当出现大量的发布、部署工作时，如果一台一台去修改，发布周期会非常长，而且这种又无聊又繁琐的工作老子早就烦的掀桌子不想干了， 而且这种大量重复的操作，一是没有效率，二是人工去做很容易出现错误。孔子曾经说过 “懒人使科技进步”（误），国外有个叫 Michael DeHaan 的 “懒人” 开发了 Ansible，他在项目中对 ansible 的介绍是：

Ansible 是一个极其简单的 IT 自动化系统。它处理配置管理，应用程序部署，云配置，临时任务执行和多节点编排 - 包括使用负载平衡器实现零停机滚动更新等微不足道的操作

Ansible 糅合了众多老牌运维工具的优点实现了批量操作系统配置、批量程序的部署、批量运行命令等功能。 仅需在管理工作站上安装 ansible 程序，配置被管控主机的 IP 信息，被管控的主机无客户端，也就是说他可以代替人工，按照你之前设定好的方式，批量的执行某些操作，后在 2015 年被 RedHat 收购，收录在 epel 源中。

Ansible 特性有：

  • 模块化：能调用特定的模块，来实现特定的功能；
  • 基于 Python 开发：由 PyYAML 和 Jinja2 三个关键模块；
  • 部署简单：不需要 agent 端；
  • 幂等性：用户对于同一操作发起的一次请求或者多次请求的结果是一致的，不会因为多次点击而产生了副作用。

二、安装 Ansible¶

Ansible 收录在 epel 源中，所以先安装 epel 源, 随后安装 ansible

安装后使用 rpm -ql ansible 查看程序文件：

  • 主配置文件：/etc/ansible/ansible.cfg
  • 主机清单：/etc/ansible/hosts
  • 主程序：ansible、ansible-playbook、ansible-doc

常用模块¶

ping 模块¶

  • 测试所有远程主机的连通性

command 模块¶

  • 在远程主机运行命令，直接调用内核显示时钟

shell 模块¶

  • 在远程主机上调用 shell 来指定命令，支持 shell 特性

copy 模块¶

  • 把文件或目录复制到远程主机

fetch 模块¶

  • 将远程主机的文件拉取到本地

cron 模块¶

  • 在远程主机上创建 crontab 计划任务

file 模块¶

  • 在远程主机上操作文件

创建链接文件 path=/tmp/fstab.l src=/etc/fstab state=link
修改属性 path=/etc/nginx/nginx.conf owner=root group=nginx mode=644
创建目录 path=/tmp/testdir state=directory

hostname 模块¶

  • 顾名思义是修改远程主机的主机名

yum 模块¶

  • 管理远程主机的 yum

service 模块¶

  • 管理服务

user 模块¶

  • 管理远程主机的账号

setup¶

  • 获取 facts，一些自带变量
