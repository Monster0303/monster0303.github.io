Docker 网络¶

虚拟网络分类¶

首先，回顾一下虚拟化中的四种网络类型。

类型 描述
隔离桥 内部网络完全封闭，只能内部通信
仅主机桥 在隔离的基础上，仅能和宿主机通信
路由桥 内网主机可以通过路由把报文转发出去，但是如果外部没有对应的路由，报文是回不来的
NAT 桥 内网主机报文通过 NAT 桥上的网关地址转发出去，回来时再通过 连接追踪 机制转发给内网主机

这篇《 一篇文章讲解什么是虚拟化和 kvm 的使用方法 》详细描述了各种网络模型的实现方法

Docker 中的网络分类¶

Docker 中也有四种网络，但是 Docker 中的网络使用了 Linux Bridge 和 namespace 机制，类似于虚拟化中的网络，但又不完全一样

这里说明 Docker 的网络分类和使用方法

类型 描述
桥网络 类似虚拟机中的网桥，默认是 docker(NAT)
共享桥网络 将 6 种名称空间内的 UTS、IPC、Network 共享，但是其他 3 个依然独立，也称之为 联盟式网络，这样在共享桥内的网络就像在同一台主机运行一样，但是 Mount、User 和 PID 依然独立
共享宿主机网络 容器直接使用宿主机的根网络名称空间，宿主机能看到网络信息，容器内也都可以看到
none 网络 无网络，就好像没有网卡的计算机设备

开放入站通信¶

如果容器内部运行了一个服务，想要对外开放，可以把端口映射到宿主机上，方法是在启动容器时，加入 -p 选项指定映射方式

-p 选项的使用格式：

(1). -p <containerPort> 将容器的指定端口，映射至主机所有地址的一个动态端口

(2). -p <hostPort>:<containerPort> 将容器的指定端口，映射至主机的指定端口

(3). -p <ip>::<containerPort> 将容器指定的端口，映射至主机指定的动态端口

(4). -p <ip>:<hostPort>:<containerPort> 将容器的指定端口，映射至主机指定的 ip 和指定的端口
