一、Cobbler 介绍¶

1. 介绍¶

  • Cobbler 是一个 Linux 服务器安装的服务，可以通过网络启动 (PXE) 的方式来快速安装、重装物理服务器和虚拟机，同时还可以管理 DHCP，DNS 等。
  • Cobbler 可以使用命令行方式管理，也提供了基于 Web 的界面管理工具(cobbler-web)，还提供了 API 接口，可以方便二次开发使用。
  • Cobbler 是较早前的 kickstart 的升级版，优点是比较容易配置，还自带 web 界面比较易于管理。
  • Cobbler 内置了一个轻量级配置管理系统，但它也支持和其它配置管理系统集成，如 Puppet，暂时不支持 SaltStack。

Cobbler官网

2. Cobbler 集成的服务¶

  • PXE 服务支持
  • DHCP 服务管理 (用来分配 ip 地址和 TFTP 的)
  • DNS 服务管理 (可选 bind,dnsmasq)
  • 电源管理
  • Kickstart 服务支持
  • YUM 仓库管理
  • TFTP(PXE 启动时需要)
  • Apache(提供 kickstart 的安装源，并提供定制化的 kickstart 配置)

3. Cobbler 工作流程¶

  1. 客户端从网络启动后，会广播包请求 DHCP 服务器
  2. DHCP 服务器（cobbler server）收到请求后发送 responese，包括其 ip 地址
  3. 客户端拿到 ip 后再向 cobbler server 发送请求 OS 引导文件的请求(pxelinux.0)
  4. cobbler server 告诉客户端 OS 引导文件的名字 (一般为 pxelinux.0) 和 TFTP 的 ip 和 port
  5. 客户端通过上面告知的 TFTP server 地址通信，下载引导文件
  6. 客户端执行该引导文件，在内存中展开，确定加载信息，选择要安装的 os，期间会再向 cobbler server 请求 kickstart 文件和 os image
  7. cobbler server 发送请求的 kickstart 和 os iamge
  8. 客户端加载 kickstart 文件
  9. 客户端接收 os image，安装该 os image

4. 使用 cobbler check 进行自检¶

提示了 7 条问题，一条一条来解决

解决方法：

    1. 安装 dhcp 并 systemctl start dhcpd 解决
    1. CentOS7 不用管，CentOS6 需要启动超级守护进程启动
    1. 其实不需要太多的 BootLoader，根据自己环境吧，可以忽略，我这里进行了升级，只是试一试，执行 cobbler get-loaders
    1. 没有检测到 rsync 的服务，启动 rsyncd.service 后解决
    1. deb 系的 Linux 发行版需要，如没有可以忽略，我这里选择忽略
    1. 提示没有安装 pykickstart，执行 yum install pykickstart 安装后解决
    1. fencing 是高可用使用的，这里不需要，忽略
