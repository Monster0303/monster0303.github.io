实验介绍¶

众所周知，LVS 支持内核级调度，功能非常强大，但是 LVS 有一个致命的缺点，就是不支持对后端的 RS 做健康状态检测，RS 如果宕机了依然会把请求调度给故障的 RS，而且 Director 节点自己也存在单点故障的隐患，那 LVS 在生产环境中如何稳定的使用呢？

上篇文章已经介绍了 Keepalived 的使用方法和配置参数，《 实现高可用集群的神器 详解 Keepalived 》

所以本文将介绍在生产环境中 LVS 的正确使用方法：使用 Keepalived 实现 LVS 高可用

在 Keepalived 的加持下，可以实现对 Director 节点双活，并支持基于 4 层或 7 层对后端的 RS 节点做健康状态检测，能根据检测结果自动踢除故障的 RS，待 RS 健康后自动加回集群，让 LVS 的功能发挥到极致。

一、配置两台 Realserver¶

安装配置 Nginx¶

在两台 Realserver 中都安装好 Nginx。

创建测试页，生产环境中两台 Realserver 的网页应该相同，这里为了测试效果故意放置不同的内容。

启动服务。

配置 IPVS¶

在两台 Realserver 中都创建配置 IPVS 的脚本文件。

如果想了解 LVS 的 DR 模型详细原理，可以先看看这篇 《 LVS 集群负载器的原理和配置 》

vim /root/realserver.sh

执行脚本，Realserver 就配置好了。

二、配置两台 Director¶

安装软件¶

两台 Director 中都安装好 nginx 和 keepalived。

使用 Nginx 作为 sorry server，创建 error server 的主页文件。

配置 Keepalived¶

在 Keepalived 配置文件中定义好 LVS 的规则，Keepalived 会自动创建 LVS 规则并检测健康状态。

配置 Director 1 中的 Keepalived。

vim /etc/keepalived/keepalived.conf

配置 Director 2 中的 Keepalived。

vim /etc/keepalived/keepalived.conf

启动服务¶

启动 Nginx 和 Keepalived 服务。

添加 DNS 解析¶

把域名绑定到两个 VIP 上。

三、测试效果¶

使用客户端连续访问域名进行测试，可以看到轮询的效果。


测试高可用¶

模拟 RS 宕机一台¶

在客户端循环每隔 1 秒访问一次集群，然后把 RS2 的 Nginx 服务停止来模拟 RS 宕机

模拟两台 RS 全部宕机¶

Keepalived 检测到两台 RS 全部宕机后，会将请求调度到之前准备的 Sorry server 上去。

模拟 Dorector 宕机一台¶

先恢复 Nginx，然后依然在客户端循环每隔 1 秒访问一次集群，然后停止 Director 2 上的 Keepalived 服务模拟 Director 节点宕机一台。
