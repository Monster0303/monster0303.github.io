什么是 Dashboard¶

Dashboard 是一个运行在 k8s 之上的 Pod，通过 ServiceAccount 账号将用户的操作代理至 k8s。

准备工作¶

1. 注意事项¶

dashboard 从 v2.0.0-beta6 版本开始，与 dashboard 相关的资源从名称空间 kube-system 改为 kubernetes-dashboard，所以部署前必须先创建好名称空间。

注意：本文使用 v2.0.0 版本进行演示，如果你安装的 dashboard 是较低版本的 v1.10，请把本文所有步骤中的命令，与名称空间相关的参数从 -n kubernetes-dashboard 改为 -n kube-system。

2. 创建证书¶

为了安全，建议将 dashboard 部署为 HTTPS，并使用互联网认证 CA 所签发的证书。

注意：这个证书是用于客户端通过 SSL 访问 dashboard 时使用，而不是作为 k8s 集群认证使用。

访问 dashboard¶

有以下几种方式访问 dashboard：

  • NodPort：把 service 的类型改为 NodePort；
  • 以 Ingress 方式访问dashboard；
  • 以 API server 方式访问 dashboard；
  • 以 kubectl proxy 方式访问 dashboard。

登录 dashboard¶

登录 dashboard 时需要一个服务账号（ServiceAccount），支持 Kubeconfig 和 Token 两种认证方式，下面一一介绍使用方法。
